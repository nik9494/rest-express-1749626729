## 🎯 Ключевые различия типов комнат

### Стандартные комнаты

**Роли:**
- ✅ Создатель = ИГРОК (не наблюдатель!)
- ✅ Все участники = игроки
- ❌ Нет организатора или наблюдателей

**Логика:**
- ✅ Создатель платит за вход как обычный игрок
- ✅ Весь призовой фонд идет победителю (100%)
- ✅ Таймер ожидания: 60 секунд
- ✅ Автоматическое создание/присоединение
- ✅ Автостарт при заполнении комнаты

**API endpoints:**
- `GET /api/v1/standard-rooms` - список комнат
- `POST /api/v1/standard-rooms` - создание комнаты
- `POST /api/v1/standard-rooms/:id/join` - присоединение
- `POST /api/v1/standard-rooms/auto-join` - автоподбор

### Hero-комнаты

**Роли:**
- ✅ Организатор (создатель) = НАБЛЮДАТЕЛЬ (не может играть!)
- ✅ Остальные участники = игроки
- ✅ Возможность присоединения как наблюдатель

**Логика:**
- ✅ Организатор НЕ платит за вход
- ✅ Организатор получает 10% от призового фонда
- ✅ 90% призового фонда идет победителю
- ✅ Таймер ожидания: настраивается организатором (30-600 сек)
- ✅ Уникальный код для присоединения
- ✅ Ручной старт игры организатором
- ✅ Возможность удаления комнаты организатором

**API endpoints:**
- `GET /api/v1/hero-rooms` - список комнат
- `POST /api/v1/hero-rooms` - создание комнаты
- `POST /api/v1/hero-rooms/:id/join` - присоединение как игрок
- `POST /api/v1/hero-rooms/:id/observe` - присоединение как наблюдатель
- `POST /api/v1/hero-rooms/:id/start` - старт игры (только организатор)
- `DELETE /api/v1/hero-rooms/:id` - удаление комнаты (только организатор)

## 🔄 Маршрутизация

### Автоматическое определение типа комнаты
- `/waiting-room/:roomId` → автоматически перенаправляет на правильную страницу
- `/waiting-room/standard/:roomId` → страница ожидания стандартной комнаты
- `/waiting-room/hero/:roomId` → страница ожидания Hero-комнаты

### Логика перенаправления
1. Пользователь переходит по `/waiting-room/:roomId`
2. Компонент `RoomRedirect` определяет тип комнаты через API
3. Автоматическое перенаправление на правильную страницу

## 🛡️ Критически важные правила

### Для Hero-комнат:
```typescript
// ПРАВИЛЬНО: организатор = наблюдатель
await storage.addParticipant({
  room_id: roomId,
  user_id: organizerId,
  is_observer: true, // КРИТИЧЕСКИ ВАЖНО!
  entry_fee: "0" // Организатор не платит
});
```

### Для стандартных комнат:
```typescript
// ПРАВИЛЬНО: создатель = игрок
await storage.addParticipant({
  room_id: roomId,
  user_id: creatorId,
  is_observer: false, // КРИТИЧЕСКИ ВАЖНО!
  entry_fee: String(entryFee) // Создатель платит
});
```
